# Инициализация структуры проекта сервиса Moderation

Этот пулл-реквест создает базовую структуру проекта для нового сервиса **Moderation**, основанную на архитектуре и блоках сервиса **B2C**. Все ключевые компоненты настроены и протестированы.

---

## 📁 Описание структуры проекта

В рамках задачи были перенесены и адаптированы следующие файлы и каталоги:

### ⚙️ Конфигурация проекта (в корне)
* **`Dockerfile`**: Настройка окружения на базе `python:3.12-slim` для запуска приложения через `uvicorn`.
* **`docker-compose.yml`**: Конфигурация контейнеров для backend-сервиса (доступен на порту **`8001`**) и PostgreSQL (`postgres:16-alpine`). Настроен healthcheck для базы данных.
* **`requirements.txt`**: Список зависимостей, идентичный B2C (FastAPI, SQLAlchemy, Alembic, pytest-asyncio и т.д.).
* **`alembic.ini`**: Конфигурация инструмента миграций Alembic с путями к `src/db/migrations`.
* **`pytest.ini`**: Настройки pytest для асинхронного тестирования и автоматического поиска модулей.
* **`env.sample` / `.env`**: Шаблон и локальный файлы переменных окружения с конфигурацией базы данных `moderation_db`.

### 🚀 Исходный код (`src/`)
* **`src/main.py`**: Точка входа приложения FastAPI. Настроены кастомные обработчики исключений для возвращения ошибок в **плоском формате**:
  ```json
  {
    "code": "ERROR_CODE",
    "message": "Error description"
  }
  ```
  Также добавлен эндпоинт `/health` для проверки работоспособности.
* **`src/config.py`**: Валидация и загрузка настроек из окружения при помощи `pydantic-settings`.
* **`src/core/exceptions.py`**: Базовые исключения проекта (например, `NotFoundException`).
* **`src/api/router.py`**: Единый роутер, в который будут подключаться роутеры будущих модулей.
* **`src/db/`**:
  * **`database.py`**: Инициализация асинхронного движка SQLAlchemy и сессии, а также зависимость `get_db`.
  * **`base.py`**: Базовый DeclarativeBase-класс `Base` и `TimestampMixin` (с полями `created_at` и `updated_at`).
  * **`__init__.py`**: Импорт `Base` и точка импорта будущих моделей для автогенерации миграций Alembic.
  * **`migrations/`**: Шаблон окружения Alembic (`env.py` и `script.py.mako`) без переноса истории миграций B2C.

### 🧪 Тестирование (`tests/`)
* **`tests/conftest.py`**: Конфигурация фикстур pytest. Сессия базы данных `test_db` изолирована при помощи механизма транзакций с **nested savepoints** (изменения от каждого теста автоматически откатываются в БД, даже если вызывается коммит).
* **`tests/test_health.py`**: Базовый тест эндпоинта `/health` для проверки корректности работы структуры приложения.

### 🤖 CI/CD (`.github/`)
* **`.github/workflows/tests.yml`**: Пайплайн для GitHub Actions. Запускается при пуше или создании **Pull Request** в ветки `main`/`master`. Разворачивает контейнер с PostgreSQL, накатывает миграции (`alembic upgrade head`) и запускает тесты (`pytest`).

## 🛠️ Инструкция по локальной проверке

1. Сборка и запуск контейнеров:
   ```bash
   docker compose up --build -d
   ```
2. Применение миграций:
   ```bash
   docker compose exec backend alembic upgrade head
   ```
3. Запуск тестов:
   ```bash
   docker compose exec backend pytest
   ```
